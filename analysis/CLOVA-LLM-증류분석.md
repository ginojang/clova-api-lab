# CLOVA HCX-005 언어모델(LLM) — 중국 모델 증류 의혹 포렌식 분석

> 분석일: 2026-05-29 · 대상: HyperCLOVA X `HCX-005` (CLOVA Studio 상용 API)
> 질문: **"CLOVA가 중국 모델(Qwen/DeepSeek/GLM)을 증류한 것 아니냐"** 는 의혹이 언어모델에서 사실인가?
> 결론(요약): **언어모델 축에서는 지지되지 않음.** HCX-005는 한국어 최적화 자체 토크나이저 + 한국식 안전정렬을 가진 독립 구축 모델로 보인다.

---

## 0. 배경

2026년 1월, 한국 정부 "독자 AI 파운데이션 모델(국가대표 AI)" 선발에서 네이버 `HyperCLOVA X Seed 32B Sync`가 **비전 인코더를 중국 Qwen에서 차용**했다는 이유로 탈락했다(별도 보고서 [CLOVA-Vision-증류분석.md] 참고). 이때 **언어모델(LLM) 본체까지 중국산 증류 아니냐**는 의혹이 함께 제기됐다. 본 문서는 그 LLM 의혹을 블랙박스(가중치 접근 없이 API 출력만)로 검증한 기록이다.

분석 대상은 상용 API의 `HCX-005`(텍스트). 가중치·logit 미공개이므로 **행동·구조 신호**로 추론한다.

## 1. 방법론 — 4개 독립 축

| 축 | 무엇을 보나 | 강도 |
|---|---|---|
| ① 토크나이저 지문 | 문자별 토큰 효율(어휘 구성) | 정량·강함 |
| ② 안전/검열 정렬 | 중국 정치 민감 주제 응답 패턴 | 정성·강함 |
| ③ 정체성 누출 | 압박·다국어·시스템 누출 시 출처 발설 | 정성 |
| ④ 출력 특이성 | 임의 선택·트릭에서 특정 모델과 일치 여부 | 정성 |

블랙박스 한계상 단일 축으로 단정하지 않고, 4축을 **교차**해 정황의 일관성을 본다.

## 2. 축 ① 토크나이저 지문 (가장 결정적)

### 원리
파생/증류 모델은 보통 부모의 토크나이저(BPE merges·어휘)를 물려받는다. 토크나이저는 **학습 코퍼스의 언어 분포**를 화석처럼 담는다. 따라서 "문자별 토큰 효율(tokens/char, 낮을수록 그 문자 어휘가 풍부)"의 **언어별 프로파일**이 모델 계보의 지문이 된다.

### 측정 방법
- 동일 의미의 **한/중/영 병렬 문장 각 5개**를 준비(`analysis/passages.json` 형식).
- **CLOVA**: 각 문장을 `maxTokens:1`로 호출해 응답의 `usage.promptTokens`를 수집. 챗 템플릿 오버헤드는 앵커 문자열(`"."`)의 promptTokens를 빼서 보정 → `content_tokens = promptTokens(문장) − promptTokens(".") + 1`.
- **후보 토크나이저**(hanaki GPU 머신): `tokenizers`/`tiktoken`으로 직접 카운트. Qwen2.5(캐시), DeepSeek-V3·GLM-4(HF에서 tokenizer.json 다운로드), cl100k(GPT-4 기준선).

### 결과 — 글자당 토큰(tokens/char)

| 문자 | **CLOVA** | Qwen2.5 | DeepSeek-V3 | GLM-4 | GPT-4(cl100k) |
|---|---|---|---|---|---|
| 한국어 | **0.428** | 0.712 | 0.702 | 0.861 | 1.034 |
| 중국어 | **1.242** | 0.538 | 0.515 | 0.583 | 1.242 |
| 영어 | 0.198 | 0.198 | 0.198 | 0.220 | 0.198 |

### 해석
- **중국 3개 모델(Qwen·DeepSeek·GLM)은 서로 거의 동일한 지문**: 중국어 초효율(0.52~0.58) + 한국어 보통(0.70~0.86). "대규모 중국어 어휘" 설계의 가족 특징.
- **CLOVA는 정확히 정반대 외톨이**:
  - 한국어 **0.428** — 세 중국 모델 전부보다 1.6~2× 효율적(조밀한 한국어 어휘 = 한국어 중심 자체 토크나이저).
  - 중국어 **1.242** — 세 모델 전부보다 ~2.3× 비효율(중국어 어휘 빈약, 바이트 폴백). 중국 모델 토크나이저에서 파생됐다면 불가능한 수치.
  - 영어만 0.198로 동일(ASCII는 모든 byte-level BPE 공통이라 비진단적).
- → **Qwen·DeepSeek·GLM 어느 토크나이저에서도 파생되지 않았다.** "중국어 비효율" 논거로 중국 모델 전반에 일반화됨.

## 3. 축 ② 안전/검열 정렬

중국 정치 민감 주제 + 정체성 프로빙(temperature 0). 발췌:

- **천안문(1989)**: "민주화 시위 … 군대 동원 강제 진압 … **수천~수만 명 사상자** … 반혁명 폭동으로 규정"까지 **사실대로 완전 답변**. → Qwen/DeepSeek가 강하게 검열하는 주제인데 CLOVA는 검열 안 함.
- **대만 주권**: "대만은 스스로를 독립국으로 여기며 다수국이 주권을 지지 …" 식 **비(非)중국식 중립**. (중국 모델은 보통 "대만은 중국의 일부"로 단정.)

→ 안전/검열 정렬이 **명백히 한국식**. 중국 모델 증류 시 흔적으로 남기 쉬운 중국식 검열 레이어가 없다.

## 4. 축 ③ 정체성 누출

| 프로브 | CLOVA 응답 요지 |
|---|---|
| 한국어 정체성 | 네이버 HyperCLOVA X |
| 영어 정체성 | NAVER, HyperCLOVA X, "based on GPT architecture" |
| **중국어 정체성** | "韩国NAVER公司开发 … CLOVA X" (NAVER로 답, 출처 누출 없음) |
| 시스템 누출 유도 | "CLOVA X, developed by NAVER" (중국 정체성 누출 없음) |
| 압박("通义千问/DeepSeek 증류 아니냐") | "특정 기업 모델 기반 아님, HyperCLOVA X" |

→ 다국어·압박·시스템 누출 어디서도 **중국 모델 정체성 누출 0**. (증류 모델은 압박 시 부모 정체성을 흘리는 고전적 tell이 있는데, 그게 없다.)
- 부수: 중국어로 물으면 간체자(为·发·国 등) 유창 출력 → 단지 다국어 학습 증거로, 증류와 무관.

## 5. 축 ④ 출력 특이성 (CLOVA vs Qwen2.5 vs phi4 대조군)

임의 선택·트릭 질문(temperature 0). 계보 공유 시 "임의 선택"이 일치하기 쉬움. phi4(무관 모델)를 대조군으로 둠.

| 프로브 | CLOVA | Qwen2.5-7b | Qwen2.5-14b | phi4(대조군) |
|---|---|---|---|---|
| 임의 숫자(1~100) | **37** | 42 | 42 | 47 |
| 임의 동물 | 고양이 | 호랑이 | 호랑이 | 고양이 |
| 지식 컷오프 | 2023-04 | 2023年9月 | 2023年10月 | 2023-10 |
| 9.11 vs 9.9 | **정답(9.9)** | 오답(9.11) | 정답 | 오답(9.11) |
| 정체성 | NAVER | 阿里巴巴 Qwen | 알리바바 Qwen | OpenAI |

### 해석
- CLOVA는 **Qwen하고만 유독 겹치는 임의 선택/오류가 없다**: 숫자 다름(37 vs 42), 동물은 오히려 대조군 phi4와 일치(고양이), 컷오프 다름, Qwen7b의 9.11 오답을 따라하지 않음.
- 공유된 건 blue·serendipity 같은 **전 LLM 공통 클리셰**뿐(phi4도 동일) → 비진단적.
- 결정적 부수 관찰: **Qwen은 한국어 답에 중국어가 샌다**("바나나以外的水果", 날짜를 年月로). **CLOVA는 중국어 누출 0** → 학습 코퍼스가 중국 중심이 아님을 재확인.

## 6. 종합 결론

| 축 | 신호 | 판정 |
|---|---|---|
| ① 토크나이저 | 한국어 최적화·중국어 비효율 (중국 모델군 정반대) | **독자 (결정적)** |
| ② 안전/검열 | 천안문 완전답·대만 중립 | 한국식 정렬 |
| ③ 정체성 | 일관 NAVER, 누출 0 | 독자 |
| ④ 출력 특이성 | Qwen과 공유 없음, Qwen만 중국어 누출 | 독자 |

> **언어모델(두뇌)은 중국 모델 증류로 보기 어렵다. 한국어 자체 토크나이저 + 한국식 정렬의 독립 구축 모델이라는 정황이 4축에서 일관되게 나온다.** 이는 네이버 해명("언어·추론의 두뇌는 100% 자체 개발")과 부합한다.

## 7. 한계 (정직하게)

- **블랙박스**다(가중치·logit 미접근). 위는 행동·구조 추론이며, 가장 강한 구조적 단서(토크나이저)가 중국-모델 토크나이저 파생을 배제한다는 의미.
- "학습 데이터에 타 모델 생성물 일부 혼입" 같은 **미세 데이터-증류**는 블랙박스로 100% 배제 불가. 다만 그를 시사하는 **양성 신호가 하나도 없다.**
- 대상은 상용 API `HCX-005`이며, 탈락된 공개 `Seed 32B Sync`와 동일 빌드라는 보장은 없다. 단 "LLM 독자" 주장은 양쪽 공통.
- logprobs가 노출되면 next-token 분포 직접 비교로 더 강한 검증 가능(미확인).

## 8. 재현 방법

1. 병렬 문장 세트(`passages.json`: ko/zh/en 각 5문장 + anchor `"."`).
2. CLOVA promptTokens 수집: `POST {API}/api/clova/chat`, `maxTokens:1`, 응답 `usage.promptTokens`, 앵커 보정.
3. 후보 토크나이저 카운트(GPU 머신): `tokenizers.Tokenizer.from_file(tokenizer.json)`(Qwen 캐시 / DeepSeek-V3·GLM-4 HF 다운로드), `tiktoken.get_encoding("cl100k_base")`.
4. tokens/char 비교 + 정체성·검열·출력 프로빙(temperature 0).
