# 멀티모달 = 지각(Perception) 프론트엔드 — 시작 설계서

> 작성일: 2026-05-29 · 상태: **착수(Phase 0)**
> 기반 문서: `GanpanAI/MY-LLM Architecture v2.2.md`, `GanpanAI/Image Pipeline.md`
> 한 줄: **멀티모달은 새 파이프라인이 아니라, 기존 `Intenter→Reasoner→Decoder`의 "지각 입력"을 텍스트 밖(이미지·오디오)으로 넓히는 것.**

---

## 0. 대원칙 — 기존 아키텍처 철학 그대로 확장

MY-LLM의 핵심 명제를 멀티모달에 그대로 적용한다:

> **"LLM은 추론하지 않는다 — 이해(번역)와 렌더만. 추론은 결정론·검증가능 엔진이 한다."**

```
[인코더: 지각/이해 — 퍼지]  →  [전통 엔진: 추론 — 결정론·검증가능]  →  [LLM/Decoder: 렌더]
```

- 비전/오디오 **인코더 = "이해 프론트엔드"의 비(非)텍스트 형제** (Intenter·번역기와 같은 계열, MY-LLM §2 "이해 프론트엔드 패밀리").
- 멀티모달이라고 거대 VLM을 들이지 않는다 — **소형 전문 인코더 + 결정론 엔진 + 검증** 원칙 유지(MY-LLM §2 "작은 전문가 여럿").
- 지각 단(段)만 확률적(퍼지)이고, 그 뒤 추론은 기존처럼 결정론(shapely/Z3/RAG). (MY-LLM §2 "지각적 공간이해: 추출 후 결정론 검증")

> 정직성 규율(NAVER 교훈, [../analysis/](../analysis/)): 외부 인코더 차용은 합법이나 license·출처 명시, "독자" 과장 금지. 자가검증 툴킷으로 우리 학습분이 실제 우리 것이 되는지 추적.

## 1. 멀티모달이 들어갈 세 진입점 (제품 게이트 통과 유력 순)

기존 파이프라인에 이미 자리가 설계돼 있다. 새 모달리티는 "+축"이 아니라 **"+지각 인코더 +핸드오프 경로"**다.

### A. 이미지 임베딩 → RAG 검색 *(가장 먼저·가장 안전)*
- 근거: `Image Pipeline.md` 열린항목 **`embed.asset_vector(mean_id FK, vector)`** — 이미지/텍스트 임베딩으로 asset 검색.
- 역할: 비전 인코더로 **엔티티 이미지 → 벡터** → 텍스트/이미지 쿼리로 유사도 검색. 기존 pgvector RAG(Reasoner Level 0~1)에 **자연 흡수**.
- 왜 1순위: 인코더 **동결, projector 불필요, 임베딩만 추출**. 기존 RAG 무영향으로 형제 추가. 간판 도메인에 즉시 수요(유사 간판/facade 검색).
- 부품: SigLIP / CLIP(이미 hanaki 캐시) / 또는 bge-visualized. 텍스트-이미지 공유 임베딩 공간 필요 시 CLIP류.

### B. 지각적 공간이해 (사진/도면 → 좌표·관계) *(간판 게이트 유력)*
- 근거: MY-LLM §2 **공간 resolver 3층** — "1.정량 좌표기하(shapely) / 2.정성 QSR(Z3) / 3.지각(VLM 프론트엔드)". 간판 "사진→배치 추출"이 명시됨.
- 역할: 사진/도면 → **비전이 좌표·바운딩박스·관계로 번역(퍼지)** → shapely/Z3가 적합성·이격·간섭을 **결정론 검증**.
- 왜 유력: 간판 facade/규격/이격은 실제 수요(MY-LLM §2 "공간 = 제품방향 게이트 통과 유력 후보"). 2D는 shapely 하나로 거의 커버.
- 검증/학습: shapely·캔버스 툴이 **검증기 = STaR/RLVR 무료 라벨러**(MY-LLM §2 캔버스 연동 논리 그대로). perception translator는 소형으로 RLVR.

### C. 멀티모달 이해→생성 (이미지+질문 → 설명/VQA) *(범용, 게이트 멀음)*
- 역할: LLaVA식 — 비전 인코더(동결) + **projector(자체 학습)** + LLM. 이미지 보고 한국어 설명/VQA.
- 왜 후순위: 범용 VQA는 현 간판 도메인 밖 발화(MY-LLM §2 "코딩/수학 게이트"와 같은 논리 — 방향 확정 시 착수). 다만 B의 지각 프론트엔드와 부품을 공유.

> **게이트 판단**: A(이미지 RAG)·B(간판 공간지각)는 현 도메인에 수요가 있어 **게이트 통과 유력**. C(범용 VQA)는 ELDA를 범용 멀티모달 어시스턴트로 키우는 방향이 확정될 때.

## 2. 아키텍처 매핑 — 기존 모듈에 어떻게 붙나

| 기존 모듈 | 멀티모달 확장 | 형태 |
|---|---|---|
| **Intenter** | 입력에 이미지/오디오 동반 시 modality 인지(텍스트 분류는 그대로) | 이산 분류 게이트(MY-LLM §1) |
| **Reasoner / Resolver** | **SpatialResolver의 지각 단** = 이미지→기하 번역 후 shapely/Z3. **RagResolver** = 이미지 임베딩 검색 추가 | `ResolveResult` union에 흡수(`spatial`·`rag` 변형) |
| **Decoder** | 이미지 출력은 이미 텍스트+사이드채널 설계됨(`Image Pipeline` SSE `images[]`). 공간결과는 **API 출력 큐 → canvas JSON**(MY-LLM §3) | 디스플레이 스트림 + API 큐 대칭 |
| **임베딩(embed.*)** | `embed.asset_vector` 신설 — 이미지 벡터 | per-mean 벡터(Image Pipeline 열린항목) |

→ 멀티모달은 **Reasoner의 resolver 형제 + embed 확장**으로 들어간다. 코어 계약(`ResolveResult` union, `EnrichedItem`)을 그대로 쓰고, **지각 인코더만 새로 추가**.

## 3. 부품 선택 + 라이선스 (초기)

| 역할 | 선택(초기) | license | 비고 |
|---|---|---|---|
| 이미지 임베딩(A) | SigLIP / CLIP-ViT(캐시됨) | Apache/MIT | 동결, 임베딩만. Qwen ViT는 "차용" 회피 위해 비채택 |
| 공간 지각(B) | 소형 VLM 또는 검출기 → 좌표 (perception translator) | 케이스별 | 출력=좌표/관계(형식언어). 엔진=shapely/Z3(우리 결정론) |
| 멀티모달 이해(C) | SigLIP(동결) + **projector(자체)** + 베이스 LLM | — | projector가 우리 소유 첫 부품 |
| 오디오 이해 | Whisper 인코더(캐시된 faster-whisper) | MIT | 음성→전사/이해, Intenter 앞단 |

> 모든 외부 가중치는 `multimodal/LICENSES.md`(추후)에 repo·커밋·license·용도(동결/파인튜닝) 박제.

## 4. 하드웨어 현실 (hanaki) — MY-LLM "소형" 원칙과 정합

| 항목 | 값 | 함의 |
|---|---|---|
| GPU | RTX 5060 Ti 16GB (Blackwell sm_120) | 임베딩 추출·projector 학습·소형 LoRA OK. 거대 VLM 사전학습 불가 |
| 캐시 | CLIP-ViT-base, faster-whisper, Qwen2.5-7B, bge-m3(ollama) | 비전·오디오 인코더 **이미 로컬** → A·B·오디오 즉시 착수 |
| torch | 시스템 없음 | Blackwell은 **cu128 휠** 필요(`--index-url .../cu128`). 임베딩/코사인 CPU 작업은 CPU torch로 충분 |

- MY-LLM의 "소형 전문가 여럿(0.5B·동결 인코더·엔진은 모델 0)" 원칙이 16GB 단일 GPU와 **정합**. 거인 확장은 비목표.

## 5. 단계 계획

- **Phase 0 — 셋업(지금)**: hanaki `~/mmenv`(cu128 torch+transformers), SigLIP/Whisper 로드 스모크, 자가검증 툴킷 정식화.
- **Phase 1 — 이미지 임베딩 RAG(진입점 A)**: CLIP/SigLIP로 `Front_Sign` 시드 애셋 임베딩 → `embed.asset_vector` 적재 → 텍스트→이미지 검색 PoC. 기존 RAG에 형제로 붙임.
- **Phase 2 — 공간 지각 MVP(진입점 B)**: 간판 사진 1장 → 좌표/바운딩박스 추출 → shapely 적합성/이격 1종 검증(MY-LLM §2 결정론 MVP 3종 중 1개). 캔버스 툴 연동 인터페이스 탐색.
- **Phase 3 — 평가**: clova-api-lab Bench를 이미지 입력 프롬프트로 확장(검색 정확도·공간 검증 통과율).
- **Phase 4 — (게이트 통과 시) 멀티모달 이해 C**: projector 학습으로 이미지→한국어 설명/VQA.

## 6. 1주차 구체 작업 (첫 스텝 = 진입점 A)

```
# (1) hanaki 멀티모달 venv
python3 -m venv ~/mmenv
~/mmenv/bin/pip install --index-url https://download.pytorch.org/whl/cu128 torch torchvision
~/mmenv/bin/pip install transformers pillow open_clip_torch psycopg2-binary pgvector

# (2) 이미지 임베딩 스모크 (진입점 A)
#  - SigLIP/CLIP 로드 → Front_Sign 시드 애셋(corpus-api /corpus/assets/...) 몇 장 임베딩
#  - 벡터 차원 확인, 텍스트 쿼리 임베딩과 코사인 → "유사 간판 검색" 동작 확인
#  - embed.asset_vector(mean_id FK, vector) 스키마 초안 (Image Pipeline 열린항목 실현)

# (3) (B 탐색) 사진 1장 → 좌표 추출 → shapely containment 1종 PoC
```

산출: `multimodal/scripts/`(임베딩·검색 PoC), `multimodal/LICENSES.md`(부품 출처표).
