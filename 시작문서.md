0. 목표

CLOVA Studio API를 전부 데모 테스트할 수 있는 프론트 콘솔을 만든다.

목적은 실서비스가 아니라:

API Key 입력
→ 모델 선택
→ 요청 파라미터 조정
→ 호출
→ 응답 확인
→ 스트리밍 확인
→ 로그 저장
→ Qwen/Ollama/OpenAI와 비교 가능한 구조 확보

이다.

CLOVA Studio는 HyperCLOVA X 기반 AI 서비스를 만들기 위한 네이버 클라우드 플랫폼의 개발 도구이며, API Key 발급 후 API 호출이 가능하다. 공식 문서상 OpenAI 호환 API도 제공된다.

8. CLOVA 호출 어댑터

server/services/clovaService.ts

export type ClovaChatMessage = {
  role: 'system' | 'user' | 'assistant';
  content: string;
};

export type ClovaChatRequest = {
  model: string;
  messages: ClovaChatMessage[];
  temperature?: number;
  topP?: number;
  maxTokens?: number;
  stream?: boolean;
};

export async function callClovaChat(req: ClovaChatRequest) {
  const startedAt = Date.now();

  const endpoint = `${process.env.CLOVA_BASE_URL}/testapp/v1/chat-completions/${req.model}`;

  const response = await fetch(endpoint, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-NCP-CLOVASTUDIO-API-KEY': process.env.CLOVA_API_KEY ?? '',
      'X-NCP-APIGW-API-KEY': process.env.CLOVA_APIGW_API_KEY ?? '',
    },
    body: JSON.stringify({
      messages: req.messages,
      temperature: req.temperature,
      topP: req.topP,
      maxTokens: req.maxTokens,
      stream: req.stream ?? false,
    }),
  });

  const rawText = await response.text();

  return {
    ok: response.ok,
    status: response.status,
    rawText,
    latencyMs: Date.now() - startedAt,
  };
}

네이버 CLOVA Studio 기존 API 예제들은 X-NCP-CLOVASTUDIO-API-KEY, X-NCP-APIGW-API-KEY 헤더를 사용하는 형태가 확인된다.

9. OpenAI 호환 모드도 준비

CLOVA Studio는 OpenAI 호환 API를 제공하며, 기존 OpenAI SDK에서 API 키, 엔드포인트, 모델명만 바꾸는 식의 사용을 안내한다.

따라서 내부 구조는 처음부터 Provider Adapter 형태로 만든다.

ProviderAdapter
  ├─ clovaLegacyAdapter
  ├─ clovaOpenAIAdapter
  ├─ openaiAdapter
  ├─ ollamaAdapter
  └─ qwenLocalAdapter

타입:

export type AiProvider = 'clova' | 'clova-openai' | 'openai' | 'ollama';

export type UnifiedChatRequest = {
  provider: AiProvider;
  model: string;
  messages: {
    role: 'system' | 'user' | 'assistant';
    content: string;
  }[];
  temperature?: number;
  topP?: number;
  maxTokens?: number;
  stream?: boolean;
};

14. 주의사항
API Key는 절대 localStorage에 저장하지 않는다.
테스트용이면 session memory만 사용한다.
실제 키는 server/.env에 둔다.
프론트에서 CLOVA API를 직접 호출하지 않는다.
요청/응답 원문은 개발 중에만 표시한다.
프로덕션에서는 로그 마스킹 필요.
15. 에이전트에게 줄 첫 작업 명령
이 문서를 기준으로 clova-api-lab 프로젝트를 생성해라.

우선 목표는 CLOVA Studio Chat Completion API를 테스트할 수 있는 React + TypeScript 프론트와 Node/Express 프록시 서버를 만드는 것이다.

1. Vite React TypeScript 프로젝트를 만든다.
2. server/index.ts Express 서버를 만든다.
3. /health API를 만든다.
4. /api/clova/chat 프록시 API를 만든다.
5. 프론트에는 System Prompt, User Prompt, Model, Temperature, TopP, MaxTokens 입력 UI를 만든다.
6. 실행 버튼을 누르면 백엔드를 통해 CLOVA API를 호출한다.
7. 응답 본문, Raw Request, Raw Response, latencyMs를 화면에 표시한다.
8. API Key는 프론트에 넣지 말고 server/.env에서 읽는다.
9. 이후 streaming, embedding, OpenAI compatible mode를 확장할 수 있도록 provider adapter 구조로 만든다.

이 방향이면 딱 좋아.
처음부터 “CLOVA 전용 테스트 콘솔”이 아니라 AI Provider Lab 구조로 만들면, 나중에 Qwen/Ollama/OpenAI/Claude까지 같은 화면에서 비교 가능함.
